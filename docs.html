Overview

The Client side of this project was built with Typescript using the React library. The Server side was built with NestJS, an express based framework which also utilizes Typescript. A UI Component library was built on the Chackra-UI library. PostgreSQL database was used for data persistence on the backend, and Axios is used to make API calls to the backend. A REST API was built to handle all endpoints. Emphasis was placed on scalability, reusability of UI components and easy understanding of how the code works.

Reason For Choice of Tech Stack

Typescript – This was chosen over plain JavaScript because of its static type checking nature. This helps to catch errors early as the project grows in size.

React – This library was chosen because of the component based model for the client side of the project.

NestJS – This library utilizes typescript and offers a structure for building scalable server side applications.

CLIENT SIDE (TSCD) APPLICATION BREAKDOWN

The Project is separated into multiple components folders. Each folder contains corresponding relevant components, as well as an index.ts file that exports all the individual components. This is for better structure and to achieve modularity.

  • Components
  • Layouts
  • Pages
  • Types
    •

Components Folder:

The components folder is made up of subfolders holding individual component variants, which can be reused to build out a page. It also makes building faster and makes scaling easier. The list of subfolders are as follows:

  • Alert
  • Button
  • Form
  • Heading
  • Icons
  • Modal
  • Spinner
  • Table
    •

Alert

This folder contains components for building an Alert. Inspiration and structure is derived from the Chackra-UI library. It has the following components:

  • AlertComponent: This displays a “success” or “error” alert when a collection or sample is created.

Button

This folder contains components for building a Button. It contains:

  • ButtonComponent: This is a more general button component to build custom button.
  • CreateCollectionButton: This is a custom button for creating a collection.

Form

This folder contains components for building a custom form. It contains:

  • FormInput: A component to add custom input to a form.

Heading

This folder contains components for building custom page headings. It contains:

  • Heading: A heading component to build custom headings for a page.
  • CollectionHeading: A custom heading component to create headings for Collection Pages.

Icons

This folder contains custom Icon components. It contains:

  • DeleteIcon: An icon component that serves as a delete button.
  • ViewIcon: An icon component that serves as a button to view the details of another component.

Modal

This folder contains custom components for building a display modal. It contains:

  • ModalComponent: A custom component for building a modal. Usually imported as “Modal” for naming convention.

Spinner

This folder contains custom components for building a spinner to be displayed while data is loading. It contains:

  • Spinner: A custom spinner component that is used application wide.

Table

This folder contains custom components for building a table on a page. It contains:

  • CollectionTableHead: A custom component for building the table head of a collection table.
  • CollectionTableRow: A custom component for building the table row of a collection table.
  • SampleTableHead: A custom component for building the table head of a sample table.
  • SampleTableRow: A custom component for building the table row of a sample table.

Layouts Folder:

The layouts folder is made up of subfolders holding individual component, which can be reused to set a layout for another component such as positioning, spacing of components e.t.c The list of subfolders are as follows:

  • ButtonLayout
  • FormLayout
  • PageLayout
  • TableLayout
    •

ButtonLayout

This folder contains a single custom component to set and edit the layout of button components and an index.ts file that exports it.

FormLayout

This folder contains a single custom component to set and edit the layout of form components and an index.ts file that exports it.

PageLayout

This folder contains a single custom component to set and edit the layout of a page and an index.ts file that exports it.

TableLayout

This folder contains a single custom component to set and edit the layout of table components and an index.ts file that exports it.

Pages Folder:

The pages folder is made up of subfolders holding JSX(TSX) component files that render the pages of the web application. It also holds the logic for creation, deletion, routing e.t.c for the web applications. The list of subfolders are as follows:

CollectionPage

This folder contains a single page component that displays an individual collection and its samples on clicking its view button.

ErrorPage

This folder contains a single page component that displays a “PAGE NOT FOUND” error when routed to a non existing page/component.

FormPages

This folder contains subfolders holding the form pages to create a collection and/or a sample. Its constituent components also hold logic for form validation, api calls with Axios, sending and managing data as well as rendering results. It contains:

  • CreateCollectionFormPage: A page component that renders a form to collect input and create a collection.
  • CreateSampleFormPage: A page component that renders a form to collect input, create a sample and assign it to its correct collection.

HomePage

This folder contains a single page component that displays the home page heading and imports(& renders) the collection table on the home page.

TablePage

This folder contains a single page component called HomeTablePage that displays the collection table for the home page.

Types Folder:

The types folder holds “.ts” files that contain strict types for samples and collections. It contains:

  • CollectionType: Exports an interface and type for collections
  • SampleType: Exports a type for samples.

** The “index.tsx” file in the source folder mounts/renders the main react component onto the “root” element (which is marked in the html).

** The “App.jsx” file in the source folder renders the pages components and adds routing for specific pages/components.

SERVER SIDE (IGNITION) APPLICATION BREAKDOWN

This server side application is divided into 2 modules which handles all of its capabilities:

  • Collections module
  • Sample module
    •

Collections Module:

This folder contains:

  • create-collection.dto.ts: A data transfer object (DTO) that defines how data will be sent to create a collection, within the “dto” folder.
  • collection.entitiy.ts: A file that helps maintain the shape of collection in the database in form of code that is easy to maintain, within the “entity” folder.
  • collections.controller.ts: Responsible for handling all incoming requests (GET, POST, DELETE) to the “/collections” endpoint and returning responses to the client.
  • collections.service.ts: Main source of business logic when a request is sent to the collections controller.

Sample Module:

This folder contains:

  • create-sample.dto.ts: A data transfer object (DTO) that defines how data will be sent to create a sample, within the “dto” folder.
  • sample.entitiy.ts: A file that helps maintain the shape of collection in the database in form of code that is easy to maintain, within the “entity” folder.
  • samples.controller.ts: Responsible for handling all incoming requests (GET, POST, DELETE) to the “/samples” endpoint and returning responses to the client.
  • samples.service.ts: Main source of business logic when a request is sent to the samples controller.

app.module.ts: Contains the initialization of the application and configuration for connection to the database.
